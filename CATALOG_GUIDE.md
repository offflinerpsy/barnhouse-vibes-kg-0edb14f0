## Как добавить новый проект

### 1. Подготовка изображений
- **Формат**: JPG или WebP
- **Разрешение**: минимум 1200x900px (соотношение 4:3)
- **Размер файла**: до 500KB (оптимизировать через TinyPNG)
- **Количество**: 3-5 фото на проект

### 2. Загрузка изображений
1. Создайте папку в `src/assets/projects/[тип-проекта]/[id-проекта]/`
2. Загрузите фотографии, главное фото назовите `main.jpg`

### 3. Добавление в каталог
Откройте `src/components/landing/Catalog.tsx` и добавьте объект в массив `houses`:

```typescript
{
  id: "bh-200",                    // Уникальный ID (тип + площадь)
  name: "Barn House 200",          // Отображаемое название
  projectType: "barn-house",       // Тип: barn-house, country, hi-tech, classic, minimalist
  projectLabel: "Barn House",      // Метка для бейджа
  area: 200,                       // Площадь в м²
  rooms: 6,                        // Общее количество комнат
  bedrooms: 4,                     // Количество спален
  bathrooms: 3,                    // Количество санузлов
  hasVeranda: true,                // Есть ли веранда
  verandaArea: 35,                 // Площадь веранды (если есть)
  price: "от $115 000",            // Цена
  images: [
    // Импортируйте изображения в начале файла:
    // import bhMain from "@/assets/projects/barn-house/bh-200/main.jpg";
    // И используйте их здесь:
    bhMain,
    bhInterior1,
    bhExterior2,
  ],
}
```

## Типы проектов (projectType)

| Значение      | Название   | Описание                              |
|---------------|------------|---------------------------------------|
| barn-house    | Barn House | Современный лофт-стиль                |
| country       | Country    | Загородный уютный стиль               |
| hi-tech       | Hi-Tech    | Минимализм с современными технологиями|
| classic       | Классика   | Традиционный архитектурный стиль      |
| minimalist    | Минимализм | Компактные функциональные решения     |

## Фильтрация

### По типу проекта
Автоматически по полю `projectType`

### По метражу
- до 50м² (small)
- 50-100м² (medium)
- 100-150м² (large)
- 150м²+ (xlarge)

Автоматически по полю `area`

## Обязательные поля

| Поле         | Тип     | Обязательно | Описание                    |
|--------------|---------|-------------|-----------------------------|
| id           | string  | ✅          | Уникальный идентификатор    |
| name         | string  | ✅          | Название проекта            |
| projectType  | string  | ✅          | Тип проекта                 |
| projectLabel | string  | ✅          | Метка для бейджа            |
| area         | number  | ✅          | Площадь в м²                |
| rooms        | number  | ✅          | Количество комнат           |
| bedrooms     | number  | ✅          | Количество спален           |
| bathrooms    | number  | ✅          | Количество санузлов         |
| hasVeranda   | boolean | ✅          | Наличие веранды             |
| verandaArea  | number  | ❌          | Площадь веранды (если есть) |
| price        | string  | ✅          | Цена                        |
| images       | array   | ✅          | Массив URL/импортов (мин 1) |

## Инструкция для ИИ

При добавлении нового проекта:
1. Создай папку `src/assets/projects/[тип]/[id]/`
2. Скопируй изображения в папку
3. Добавь импорты в начало `Catalog.tsx`
4. Добавь объект в массив `houses`
5. Проверь что все поля заполнены корректно
6. Убедись что `id` уникален

При удалении проекта:
1. Удали объект из массива `houses`
2. Удали неиспользуемые импорты
3. Удали папку с изображениями

## Важные заметки

- Изображения должны быть оптимизированы для веба
- Первое изображение в массиве используется как превью в каталоге
- При отсутствии веранды поле `verandaArea` можно не указывать
- Цена указывается в формате "от $XX XXX"
